# FASM AutoDocer

Инструмент для автоматической генерации GitHub Wiki из исходного кода на FASM. Написан полностью на FASM. Использует формальные методы парсинга для извлечения документации из комментариев.

## Технические характеристики

* **Язык реализации:** FASM (x86/x64).
* **Алгоритм парсинга:** LL(1) рекурсивный спуск.
* **Лексер:** На базе таблицы классов символов (256 вхождений) и автомата состояний.
* **Зависимости:** Отсутствуют (Self-hosted).
* **Формат вывода:** Markdown-структура для репозитория GitHub Wiki.

## Синтаксис разметки

Документирование осуществляется через блоки `; @[...]` перед метками.

```fasm
; @[proc]
; .name: имя_процедуры
; .desc: краткое описание
; .in:   входные параметры (регистры/стек)
; .out:  выходные параметры
; .note: дополнительные примечания
label_name:

```

## Структура программы

1. **Lexer:** Чтение байтового потока, классификация символов, выдача токенов.
2. **Parser:** Синтаксический анализ потока токенов согласно грамматике LL(1). Построение IR (Intermediate Representation) в памяти.
3. **Generator:** Обход IR и формирование файлов:
* `Home.md` — оглавление.
* `_Sidebar.md` — навигация.
* `*.md` — страницы модулей.



## Сборка и запуск

1. Сборка:
`fasm autodocer.asm`
2. Генерация документации:
`./autodocer <input_file.asm> <output_directory>`

## Цель проекта

Автоматизация поддержки актуальной технической документации для низкоуровневых проектов (ОС, драйверы) без использования сторонних высокоуровневых сред исполнения.
